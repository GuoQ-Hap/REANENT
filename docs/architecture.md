# PMC 库存供应链智能体架构

## 总体分层

```text
用户 / 业务系统触发
  -> 接口层
  -> Agent 编排层
  -> 规则工具层
  -> 数据连接层
  -> 治理与审计层
  -> 结果输出 / 人工确认 / 反馈闭环
```

## 1. 接口层

接口层负责接收外部请求，并把请求统一交给 Agent 编排服务。

- CLI：用于本地验证、调试和快速演示。
- HTTP API：用于后续接入前端页面、内部系统或调度任务。
- 未来前端：库存控制塔、风险列表、Agent 侧栏、异常 Case 面板都可以调用同一套编排服务。

## 2. Agent 编排层

Agent 编排层负责理解问题、拆解任务、选择工具、汇总结果。

- 任务分类：识别控制塔、断货追因、发货验证、采购验证、周度计划、异常 Case、规则问答等请求。
- 计划生成：把用户请求拆成可执行步骤，例如先取库存快照，再做复算，再解释差异。
- 工具调用：按计划调用库存、发货、采购、知识库、Case 等工具。
- 结果组织：把规则计算结果、原因链、建议动作、人工确认点整理成业务可读输出。

当前版本已经把意图识别抽象为模型客户端，由模型基于语义和上下文返回结构化判断；计划生成仍保持确定性，方便测试和稳定落地。

## 3. 规则工具层

规则工具层只做可复算、可解释的业务计算。数量、日期、金额和差异不交给大模型直接生成。

首期工具包括：

- 库存控制塔：识别高风险断货、冗余库存、库存异常、物流异常、采购异常。
- 库存健康计算：判断断货预警、冗余库存、库存异常。
- 断货追因：按 FNSKU / SKU 输出 FBA 可售、未来预测、在途、交期、采购和发货窗口。
- 发货验证：复算基础发货量、发货修正量、发货参考量，并解释差异。
- 采购验证：复算基础采购量、采购修正量、采购参考量，处理 MOQ、箱规、可拼采购、需求过低等规则。
- 周度发货计划：生成发货时间、物流渠道、发货仓库、整箱/半箱数量、待配货、异常原因等草稿。
- 异常 Case：生成异常原因、处理建议、责任角色、状态和反馈记录。
- 知识库查询：解释字段口径、SOP、备货规则、发货/采购计算规则。

## 4. 数据连接层

数据连接层负责把 ERP、FBA、WMS、数据仓库、Excel 规则表等数据统一成工具可用的数据结构。

首期重点数据源：

- `ads_lingxing_all_warehouse_new_v1`：库存监控主宽表。
- `dim_inventory_forecast_v1`：采购侧未来库存预测。
- `dim_inventory_forecast_v1_fh`：发货侧未来库存预测。
- `ads_lingxing_sc_sales_daily_new`：销售日报、实际销量、广告与促销数据。
- `dwd_lingxing_fba_warehouse_detail`：FBA 明细库存。
- `dwd_lingxing_inventory_details`：国内仓库存明细。
- `temp_lingxing_stocking_rules`：备货规则。
- Excel 补充规则：备货规则、起订量、箱规、供应商、物流规则。

当前实现使用内存示例数据，后续通过 connector 接入真实数据源。

## 5. 治理与审计层

治理与审计层负责防止系统越权、误操作和黑箱建议。

- 权限边界：按 PMC、采购、物流、运营、管理层区分角色权限。
- 数据边界：按店铺、国家、销售部门、负责人控制可见数据范围。
- 人工确认：高风险动作必须人工确认，首期只生成草稿和建议。
- 审计记录：保留建议生成、人工修改、导出、确认、关闭等操作记录。
- 版本记录：保留备货规则、发货计划、采购建议的版本。
- 工具预算：限制单次任务的工具调用次数，避免无边界探索。

## 大模型演进路径

当前版本把意图识别交给模型客户端，把计划生成和工具调用保持为可测试的确定性逻辑。

后续接入大模型后，推荐演进为：

```text
用户请求 + 对话上下文 + 工具 Schema
  -> 大模型识别意图和判断下一步
  -> 外层程序执行工具
  -> 工具结果回填给大模型
  -> 大模型继续调用工具或生成最终解释
```

分工原则：

- 大模型负责理解问题、追问、解释原因链、组织建议。
- 规则工具负责计算库存数量、发货量、采购量、日期、金额和差异。
- 编排服务负责权限、工具调用、状态管理、审计和验证。
- 人工确认负责关键动作闭环，例如采购建议确认、发货计划确认、异常 Case 关闭。

## 首期业务里程碑

- P0，第 1-2 周：需求与规则梳理，确认字段口径、规则文档和现有前端计算逻辑。
- P1，第 3-5 周：规则工具后端化，完成库存健康、发货复算、采购复算、库存核对工具。
- P2，第 6-8 周：Agent 原型，支持单 SKU 追因、风险摘要、发货/采购解释。
- P3，第 9-11 周：控制塔与 Case，完成库存控制塔、风险列表、异常 Case 流程。
- P4，第 12-14 周：周度计划闭环，支持计划草稿、导出、人工确认和反馈记录。
- P5，第 15-16 周：评估与优化，跟踪采纳率、命中率、复算一致率和预测偏差。
