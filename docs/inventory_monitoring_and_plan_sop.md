# 库存监控分析与计划管理执行 SOP

> 来源：飞书文件《库存监控分析与计划管理.pdf》预览文本提取。  
> 用途：知识库沉淀、流程培训、跨部门执行对齐。  
> 注意：文中的 `X号`、`XX小时`、`5%`、`7天`、`80%上架可发` 等阈值，需要结合公司组织分工、系统能力、产品属性、物流模式确认后固化。

## 0. 总原则

库存监控分析要解决三件事：

1. 看断货：提前识别哪些产品即将断货、已经断货、断货损失多大。
2. 看冗余：识别哪些产品库存过多、库龄过高、未来成本是否高于清货/弃置。
3. 看何时补货：判断当前是否需要发货、采购，以及什么时候触发。

核心数据：

| 指标 | 含义 | 用途 |
|---|---|---|
| 库龄 | 现有存货的历史周转表现 | 判断冗余、仓储费压力、是否清货/弃置 |
| 可售天数 | 现有存货未来能卖多久 | 判断冗余、库存健康度 |
| 补货倒计时 | 现有存货能否覆盖交付周期和安全库存 | 判断断货风险、补货时点 |

跨部门职责：

| 角色 | 主要职责 |
|---|---|
| CEO/管理层 | 看问题、做决策、推动流程/组织/能力优化 |
| 运营 | 输出预估销量，监控销量预实，根据库存调整销售节奏 |
| 计划 | 监控预估销量和备货规则变化，统计断货/冗余损失，制定采购计划和发货计划 |
| 采购 | 及时下单，缩短交期，提高准交率，反馈跟单结果 |
| 物流 | 保证全链路库存准确，及时出货，反馈物流跟单结果 |
| 质量 | 管理质检、入库、返工、放行、异常纠正预防 |

## 1. 库存监控分析 SOP

### 1.1 输入

- 《预估销量表》
- 《库存监控分析表》
- 《库存分类账》或全仓库存明细
- FBA/海外仓/本地仓库存
- FBA在途、海外仓在途、本地仓在途
- 已做发货计划但未出库数据
- 已做采购计划但未入库数据
- 采购订单列表、发货单、物流单、接收报告
- 库龄报告、仓储费数据、历史销量与营销计划

### 1.2 操作步骤

1. 按 SKU/MSKU 维度汇总全链路库存。
2. 将库存拆成分段库存，例如 FBA在仓、FBA在途、海外仓、海外仓在途、本地仓、本地仓在途/采购在途。
3. 计算每一段库存对应的可售天数。
4. 计算库龄，必要时按入库批次区分。
5. 计算补货倒计时，用于判断是否需要催货、发货、下采购单。
6. 对断货风险、冗余风险、补货需求进行分级。
7. 将异常 SKU 输出到待办，并关联到后续流程：营销调整、发货计划、采购计划、物流跟单、采购跟单、清货/弃置。

### 1.3 判断规则

#### 库龄

1. 1-180天：通常按常规销售节奏管理。
2. 181-270天：进入重点观察区，关注未来仓储费和利润。
3. 271天以上：超期仓储费压力显著增加，通常需要重点评估清货、批量清货或弃置。
4. 是否清货/弃置不只看库龄，还要看未来利润、现金流影响、产品属性和可售天数。

#### 可售天数

1. 可售天数过短：可能断货。
2. 可售天数过长：可能冗余。
3. 可售天数必须结合产品属性、生命周期、季节、物流模式、仓段判断。
4. 不能只看 FBA 库存；要看全链路库存，防止“一边断货、一边冗余”。

#### 补货倒计时

基础公式：

```text
补货倒计时 = 库存可售天数 - 预估交付天数 - 安全库存天数
```

安全库存天数：

```text
安全库存天数 = 销售安全库存天数 + 物流安全库存天数 + 生产安全库存天数
```

判断：

1. 补货倒计时 > 0：当前尚未到补货点。
2. 补货倒计时 <= 0：需要进入补货/催货/发货/采购判断。
3. 补货倒计时不是“补多少货”的公式，只决定“什么时候补”。

### 1.4 补货倒计时分级动作

| 指标 | 动作 | 说明 |
|---|---|---|
| 补货倒计时1 | 催 FBA 在途 | 判断 FBA 在途是否能接上 |
| 补货倒计时2 | 发 FBA | 判断是否从海外仓/本地仓发到 FBA |
| 补货倒计时3 | 催海外仓在途 | 判断海外仓在途是否能接上 |
| 补货倒计时4 | 发海外仓 | 判断是否从本地仓/供应端发到海外仓 |
| 补货倒计时5 | 催本地仓在途 | 判断本地仓在途是否能接上 |
| 补货倒计时6 | 下采购单 | 判断是否触发采购计划 |

### 1.5 输出

- 断货风险清单
- 冗余库存清单
- 需要发货清单
- 需要采购清单
- 需要催货清单
- 需要清货/弃置清单
- 待办事项与责任人

## 2. 安全库存设置 SOP

### 2.1 定义

安全库存不是“低于 60 天就发货”。安全库存是为了抵御销量不稳定、物流不稳定、生产不稳定而保留的必要最低库存。

做计划时，应在安全库存被消耗之前，让下一批货到达。

### 2.2 设置步骤

1. 明确产品所处阶段：新品、成长期、成熟期、衰退期。
2. 判断销量稳定性：畅销、平销、滞销、季节性、销售季前/季后。
3. 判断物流稳定性：快船/慢船、渠道历史时效、月份波动、目的仓。
4. 判断生产稳定性：供应商稳定性、工艺复杂度、是否换供应商、是否一品多源。
5. 将安全库存拆分为销售安全库存、物流安全库存、生产安全库存。
6. 将结果维护到《备货规则表》。

### 2.3 增减逻辑

建议增加安全库存的情况：

- 新品
- 慢船
- 旺季
- 销售季前
- 多渠道/多店铺
- 换供应商
- 复杂工艺
- 销量不稳定
- 物流时效波动大
- 生产不稳定

建议降低安全库存的情况：

- 成熟期稳定产品
- 衰退期产品
- 滞销品
- 淡季
- 合仓
- 一品多源
- 简化工艺
- 快船且物流稳定

### 2.4 销售安全库存计算

销售安全库存用于解决特殊情况下超卖的问题，由可超卖比例决定。

示例：

```text
日销 = 100件
物流天数 = 60天
可超卖比例 = 20%
最低库存量 = 100 * 60 * (1 + 20%) = 7200件
销售安全库存量 = 100 * 60 * 20% = 1200件
销售安全库存天数 = 1200 / 100 = 12天
```

### 2.5 物流安全库存计算

1. 收集同一物流渠道、同一月份的历史物流天数。
2. 统计各物流天数的出现概率。
3. 根据希望覆盖的准交概率选择安全天数。
4. 判断安全天数是放在仓库库存中，还是加入物流天数中。

区别：

- 放入安全库存：库存实际放在仓库。
- 加入物流天数：库存可能体现为在途。

## 3. 冗余库存处理 SOP

### 3.1 输入

- 库龄
- 可售天数
- 当前售价
- 采购成本、头程成本、配送费、佣金、营销成本
- 未来基础仓储费
- 未来超期仓储费
- 退货处理费、二次上架费用、货损成本
- 清货新增营销成本
- 移除/弃置成本
- 批次成本和批次库龄

### 3.2 判断步骤

1. 按批次计算当前库存成本，不同采购批次、物流批次、入库批次要分开。
2. 计算正常销售状态下的未来成本和利润。
3. 计算清货销售状态下的未来成本和利润。
4. 计算移除/弃置状态下的回款或损失。
5. 对比三种方案：正常销售、增加营销投入清货、批量清货/弃置。
6. 输出建议：慢慢卖、正常清货、批量清货、弃置。

### 3.3 启动清货判断

满足任一条件即可启动清货评估：

```text
正常销售状态下的成本增量 >= 清货销售状态下的成本增量
```

或：

```text
正常销售状态下的利润 <= 清货销售状态下的利润
```

清货成本增量包括：

- 新增折扣
- 活动费用
- 广告花费
- 佣金变化
- 未来基础仓储费
- 未来超期仓储费
- 配送费
- 退货成本
- 计提跌价准备
- 单位货损成本

### 3.4 批量清货/弃置判断

判断公式：

```text
回款 = 成交价 - 成本增量
成本增量 = 新增总仓储费 + 新增营销成本 + 配送费 + 佣金 + 退货处理费 + 货损成本 ...
```

当：

```text
回款 <= -移除成本
```

或：

```text
销售利润 <= 移除利润
```

即可考虑批量清货/弃置。

原则：

> 批次库存是否移除，看是否还有回款，不要被沉没成本误导。

### 3.5 输出

- 《利润测算表》
- 《弃置产品列表》
- 《清货产品列表》
- 营销调整建议
- 移除/弃置申请
- 纠正预防措施

## 4. 发货计划 SOP

### 4.1 目的

减少断货和冗余损失，缩短发货时间，降低发货成本。

### 4.2 输入与输出

```text
输入：营销计划《预估销量表》、库存监控分析表
输出：发货计划《发货计划表》
```

### 4.3 正常活动节点

1. 创建发货计划。
2. 装柜试算。
3. 本地仓调拨。
4. 创建 FBA 货件。
5. 输出发货计划表。

### 4.4 执行步骤

1. 运营每月 `X号` 输出《销量预估表》。
2. 运营通过《库存监控分析表》校核断货风险1-6，以及在途批次是否能在断货前接上。
3. 对于中度/严重断货风险，且不能断货的产品，先下调预估销量，使风险降为轻微或无断货风险，再修正《销量预估表》。
4. 计划部门在 `24小时内` 对修正后的《库存监控分析表》二次复核。
5. 如仍有断货风险且不能断货，计划部门与运营二次确认。
6. 对确认无误的 SKU，计划部门当日按基础补货公式创建发货计划，填写发货数量和发货仓库。
7. 如需整柜出运，进行整柜试算，并在当日完成。
8. 根据整柜试算、分仓需求、存货数量判断是否需要增减发货数量。
9. 若增减数量，重新通过《库存监控分析表》试算。
10. 若调整后仍有断货风险或利润不达标，先内部审核是否存在替代方案。
11. 内部审核通过后，提交运营当日审核，判断是否调整发货计划。
12. 若不调整发货计划，则判断是否需要调整营销计划。
13. 输出最终《发货计划表》。

### 4.5 异常判断

#### 目的地异常

- 发到 FBA：需要创建货件。
- 发到海外仓：不需要创建 FBA 货件，但可能需要付款流程。

#### 数量异常

可能增加计划量的原因：

- 分仓
- 物流 MOQ
- 物流阶梯价
- 装柜尾数
- 整柜装载要求

可能减少计划量的原因：

- 缺货
- 装柜尾数
- 实际可发库存不足

处理：

1. 若数量变化带来断货或冗余风险，重新试算《库存监控分析表》。
2. 必要时调整营销计划。

#### 时间异常

1. 如果计划发货时间无法满足标准交期，需要调整发货计划。
2. 如果有断货风险，需要同步评估是否调整营销计划。

### 4.6 审核规则

审核本质上只审核异常。

计划经理/总监需要审核以下情况：

1. 发货计划量增减幅度超过 `5%`。
2. 计划到货时间或发货时间与标准时间有偏差。
3. 选择加急交期。
4. 调整会造成断货或冗余风险。

审核时必须输出：

- 差异原因
- 是否可通过流程减少后续同类差异
- 是否需要调整营销计划
- 是否需要调整发货计划

## 5. 发货流程 SOP

### 5.1 目的

缩短发货时间，减少发货错误，降低发货成本。

### 5.2 输入与输出

```text
输入：发货计划《发货计划表》
输出：发货出库《发货单》、头程物流单
```

### 5.3 正常活动节点

1. 创建发货单。
2. 配货。
3. 拣货。
4. 装箱。
5. 贴标。
6. 发货出库。
7. 装柜试算。
8. 本地仓调拨。
9. 创建 FBA 货件。
10. 物流下单/订舱。

可选节点：

- 出库质检
- 称重
- 红外测量尺寸

### 5.4 执行步骤

1. 物流专员在发货计划提交后 `XX小时内` 创建《发货单》。
2. 在发货单中明确：发货数量、始发仓、目的仓、发货渠道、发货时间、标准到货时效。
3. 若始发仓库存不足，先配货；配货方式包括加工、调拨、采购。
4. 配货需在 `XX小时内` 完成。
5. 若配货时间会造成断货，同步断货时间和控销数量给运营与计划。
6. 若发到 FBA，由物流在 `XXX小时内` 创建货件。
7. 判断是否分仓：分目的仓、分始发仓、多仓发货。
8. 分目的仓不够 `5箱` 的产品，先看能否拆小箱；不能拆时再考虑增加数量。
9. 需要整柜的产品，完成装柜试算，装柜量不得低于公司标准立方数。
10. 根据配货、分仓、装柜结果判断是否需要增减数量。
11. 对数量或时间调整的发货单，物流经理在 `XX小时内` 审核。
12. 审核通过后，物流专员当日完成物流下单/订舱。
13. 与物流商确认发货时间、发货数量等信息。
14. 若物流单无法满足发货需求，重新修订发货单。
15. 若物流单与发货单一致，归档发货单和物流单。

### 5.5 异常判断

#### 数量异常

1. 是否有货；没货则先配货。
2. 是否存在绑定店铺或 FNSKU 的库存；如存在，判断是否需要解绑。
3. 是否因分仓、物流 MOQ、阶梯价、尾数增加发货量。
4. 是否因缺货、装柜尾数减少发货量。
5. 若减少发货量有断货风险，则调整营销计划和发货计划。

#### 始发仓/目的仓异常

1. 发到 FBA/Wayfair：需要创建货件。
2. 发到海外仓：不创建 FBA 货件。
3. 同类型仓库是否需要分仓，如 DE、FR、IT、ES，或美国 FBA 分仓。
4. 多仓发货时，需要拆行。
5. 工厂直发时，系统可能需要先自动入库再扣减库存。

#### 时间异常

1. 发货无法满足标准时间时，需要调整。
2. 若有断货风险，需要调整营销计划和发货计划。

#### 发货要求异常

1. 是否需要贴标、换标、装箱、拆箱。
2. 是否需要计重和测量尺寸。
3. 是否需要横放、竖放、防压坏。

### 5.6 审核规则

物流经理/总监审核以下情况：

1. 发货量与计划量增减幅度超过 `5%`。
2. 计划到货时间/发货时间与标准时间有偏差。
3. 选择加急交期。

审核输出：

- 差异原因
- 是否偶发
- 是否可通过流程减少同类问题
- 是否需要反馈运营调整营销计划

## 6. 出库流程 SOP

### 6.1 目的

缩短出库时间，减少发货错误，降低发货成本。

### 6.2 输入与输出

```text
输入：发货《发货单》
输出：出库《出库单》
```

### 6.3 正常活动节点

1. 创建出库单。
2. 拣货。
3. 装箱。
4. 贴标。
5. 发货出库。

可选节点：

- 出库质检
- 称重
- 红外测量尺寸

### 6.4 异常判断

#### 数量异常

1. 实际装柜可多装，是否增加发货数量。
2. 胀箱导致实际装不下，是否减少发货数量。

#### 时间异常

1. 是否因质量问题返工，导致延期出库。
2. 是否因货到晚，导致延期出库。

### 6.5 关键注意事项

1. 供应商直发 FBA 或海外仓时，可不经过拣货。
2. 下采购订单时可提前要求装箱、贴标，使入库时已完成装箱贴标。
3. 从营销计划、发货计划、发货单到出库单，任何数量或时间变化都可能导致断货或冗余风险。
4. 建议在出库流程中反馈运营调整营销计划，以提高流程效率。

## 7. 物流跟单 SOP

### 7.1 目的

及时跟进物流进度，将异常反馈给运营，减少断货和冗余损失。

### 7.2 输入与输出

```text
输入：发货出库《发货单》
输出：接收入库《接收单》或入库通知《入库通知单》
```

### 7.3 正常活动节点

1. 开船。
2. 到港。
3. 提柜。
4. 约仓。
5. 送仓。
6. 入库。
7. 上架。

### 7.4 异常判断

#### 目的地

- 发到 FBA：需要跟踪到数量的 `80%` 上架可发。
- 发到海外仓：跟踪到接收入库或入库通知。

#### 数量异常

1. 接收/上架/入库数量是否与发货单一致。
2. 若大量丢件，通知运营，并及时关闭在途数量。

#### 时间异常

1. 若延期且有断货风险，至少提前 `7天` 反馈运营。
2. 判断是否需要加急物流、调整营销计划、调整发货计划。
3. 加急不是必选项，需结合利润测算判断。

### 7.5 审核规则

物流经理/总监审核以下情况：

1. 接收/入库数量增减幅度超过 `5%`。
2. 预估到货时间与标准时间相比，延期超过安全库存天数。

审核时需判断：

- 偶然现象，还是物流渠道/路线/天数设置问题。
- 是否安全库存天数设置不合理。
- 是否需要纠正预防措施。

## 8. 采购计划 SOP

### 8.1 目的

减少断货和冗余损失，缩短采购下单时间，降低采购成本。

### 8.2 输入与输出

```text
输入：营销计划《预估销量表》
输出：采购计划《采购计划表》
```

### 8.3 正常活动节点

1. 创建采购计划。
2. 校核断货风险。
3. 校核冗余风险。
4. 判断 MOQ、阶梯价、交付时间。
5. 输出采购计划。

### 8.4 异常判断

#### 数量异常

1. 采购 MOQ、阶梯价可能导致计划量增加。
2. 一般很少出现减少采购计划量的情况。
3. 若 MOQ/阶梯价导致冗余风险，需要重新试算利润和库存。

#### 时间异常

1. 标准交付时间无法满足计划时，需要调整采购计划。
2. 若有断货风险，需要调整营销计划。

#### 目的地异常

1. 若生产后直接发 FBA，需要创建货件并提前贴标。
2. 若进入本地仓/海外仓，按对应库存节点计算补货倒计时。

### 8.5 审核规则

计划经理/总监审核以下情况：

1. 采购计划量增加幅度超过 `5%`。
2. 计划到货时间/发货时间与标准时间有偏差。
3. 选择加急交期。

注意：

- 是否加急物流或加急生产，建议前置到营销计划中判断，因为会影响利润。
- 大额采购或超预算采购，本质也应由销售计划传递下来，审批应前置。

## 9. 采购下单 SOP

### 9.1 目的

缩短下单时间，减少下单错误，降低采购成本。

### 9.2 输入与输出

```text
输入：采购计划《采购计划表》
输出：采购下单《采购订单列表》
```

### 9.3 正常活动节点

1. 创建采购订单。
2. 提交采购订单。
3. 确认采购订单。
4. 召开产前会议。

### 9.4 异常判断

1. 送货地址是否需要修改；如修改，注明原因和新增费用。
2. 采购 MOQ、阶梯价是否导致采购量变化。
3. 下单时是否无法满足标准时间；必要时更换供应商。
4. 若有断货风险，需要调整营销计划和采购计划。
5. 是否存在绑定店铺/FNSKU 等限制。

### 9.5 供应商状态检查

供应商状态：

- 合格
- 不合格：未通过审核
- 停用：合格但不能下采购单
- 冻结：不可采购，不可付款

供应商产品状态：

- 启用
- 未启用：价格或样品未通过
- 锁定：通过但不能下单

### 9.6 审核规则

采购经理/总监审核以下情况：

1. 采购量与计划量增减幅度超过 `5%`。
2. 预估到货时间与标准时间有偏差。
3. 选择加急交期。

审核输出：

- 差异原因
- 是否供应商问题
- 是否流程问题
- 是否需要调整采购计划/营销计划

## 10. 采购跟单 SOP

### 10.1 目的

及时跟进采购生产进度，将异常反馈给运营，减少断货和冗余损失。

### 10.2 输入与输出

```text
输入：采购下单《采购订单列表》
输出：采购入库《入库单》
```

### 10.3 正常活动节点

1. 下单。
2. 回签。
3. 备料。
4. 生产。
5. 质检。
6. 送货。
7. 入库。

### 10.4 异常判断

#### 目的地异常

1. 是否需要修改送货地址。
2. 如修改，注明修改原因和新增费用。

#### 数量异常

1. 入库数量是否与采购单一致。
2. 如果入库数量低于采购计划量，且存在断货风险，需要通知运营。

#### 时间异常

1. 若延期且有断货风险，至少提前 `7天` 反馈运营。
2. 输出预计延期天数、影响 SKU、影响数量、预计销售损失。

### 10.5 审核规则

采购经理/总监审核以下情况：

1. 入库数量增减幅度超过 `5%`。
2. 预估入库时间与标准时间相比，延期超过安全库存天数。

审核时需判断：

- 是否偶然现象。
- 是否供应商排单、原材料、辅料、批量质量问题。
- 是否需要加急生产、替代供应商、备料调整。
- 是否需要纠正预防措施。

## 11. 质检入库 SOP

### 11.1 目的

缩短质检入库时间，提高问题产品和问题图纸的拦截数量。

### 11.2 输入与输出

```text
输入：入库通知《入库通知单》
输出：采购入库《质检单》《入库单》
```

### 11.3 正常活动节点

1. 质检。
2. 入库。
3. 退货。
4. 换货。

### 11.4 异常判断

#### 数量异常

1. 质检合格品数量是否与采购单一致。
2. 若合格数量低于采购订单量，通知采购跟单。
3. 若存在断货风险，反馈运营控销。

#### 质量异常

1. 出现批量不合格时，判断是否轻微问题。
2. 轻微问题可走风险放行 RRA。
3. 需要返工/换货时，做好时效管理。
4. 示例时效：`3小时` 出返工方案，`24小时` 完成返工。

#### 时间异常

1. 质检时间或换货时间超过标准时间，需要升级处理。
2. 明确库区库位在哪个环节确认。
3. 合格品贴合格标签，不合格品贴不合格标签。

### 11.5 审核规则

质量经理/总监审核所有质量异常，输出：

- 偶然现象还是原辅料/制程/工艺单问题。
- 纠正措施。
- 预防措施。
- 问题关闭时间。

## 12. 移除弃置 SOP

### 12.1 目的

减少移除弃置损失，避免无效库存持续吞噬利润和现金流。

### 12.2 输入与输出

```text
输入：移除弃置申请《利润测算表》《弃置产品列表》
输出：移除出库《出库单》
```

### 12.3 正常活动节点

1. 利润测算。
2. 制定降本方案。
3. 移除出库。

### 12.4 异常判断

#### 弃置地点

1. FBA。
2. 海外仓。
3. 本地仓。

不同地点的成本和利润测算不同。

#### 成本异常

1. 当前配送成本、仓储成本、物流成本、采购成本是否与真实成本有偏差。
2. 有偏差则修正。
3. 无偏差则判断是否有降本方案，减少弃置数量或取消弃置。

#### 时间异常

弃置时间超过标准时间，需要升级处理。

### 12.5 审核规则

运营总监审核所有异常：

- 判断是广告、活动、价格、备货等哪个环节的问题。
- 按问题类型制定纠正预防措施。

CEO 审批：

- 弃置损失超过公司设定金额时，需要 CEO 审批。

## 13. 周度跟进和月度复盘 SOP

### 13.1 周度跟进表单

至少包含：

1. 营销周度监控表。
2. 采购跟单表。
3. 物流跟单表。
4. 库存监控分析表。

### 13.2 周会重点

1. 区分结果指标和过程指标。
2. 区分考核指标和关注指标。
3. 不要把所有指标都拿来考核。
4. 对断货和冗余进行归因。
5. 所有问题落到待办、流程优化、数字化建设、组织建设、培训计划。

### 13.3 采购周度汇报结构

采购部门每周汇报：

1. 当前在途 SKU 数量、在途批次数量。
2. 上周延期交付 SKU 数量。
3. 延期交付数量、平均延期天数。
4. 造成断货风险的 SKU 数量。
5. 预计断货数量和销售额损失。
6. 上周预警延期交付 SKU 数量。
7. 加急交付单量、准交数量、涉及产品、加急原因。
8. 延期原因分类：排单、原材料、辅料、批量质量问题、其他。
9. 针对延期导致断货风险的处理方式。
10. 是否有纠正预防措施，问题是否关闭。

### 13.4 问题拆解方法

按三层拆解：

1. 表象：直观看到的结果，如断货、冗余、延期。
2. 真相：问题根源，如超卖、低卖、补晚、补少、物流延期、生产延期。
3. 法相：流程体系问题，如发货计划、采购计划、销售计划、物流跟单、采购跟单是否缺失或失效。

常见问题归因：

1. FBA/FBM 即将断货，但在途接不上。
2. FBA/FBM 断货，但全链路库存很多。
3. 好卖时货没到，不好卖时货到了。
4. 缺少数据监控，库存不知不觉过多。
5. 预估销量过高导致库存过多。
6. 预估销量过低导致断货。
7. 物流延期。
8. 生产延期。
9. 补晚了、补少了。
10. 补早了、补多了。
11. 清货过头导致断货。
