# 断货/冗余根因规则判断流程

本文档记录当前代码中的断货/冗余根因判断逻辑。根因不再使用旧兜底项，例如 `FBA前端可售不足`、`后端库存/长库龄冗余`、`有在途但未赶上断货窗口`。

## 总入口

```mermaid
flowchart TD
  A["SKU 命中风险"] --> B{"是否命中断货风险？"}
  B -- "是" --> C["进入断货根因规则树"]
  B -- "否" --> D{"是否命中冗余风险？"}
  D -- "是" --> E["进入冗余根因规则树"]
  D -- "否" --> F{"是否仅命中数据/历史异常？"}
  F -- "是" --> G["输出：数据或历史异常标记"]
  F -- "否" --> H["不输出根因"]
```

## 阈值口径

| 判断项 | 口径 |
|---|---|
| 超卖/低卖对比基准 | 只对比 `2个月前的预测值` |
| 爆/旺超卖或低卖阈值 | 偏差必须大于 `20%`，`20%以内` 不算 |
| 平/滞超卖或低卖阈值 | 偏差必须大于 `10%`，`10%以内` 不算 |
| 月度销量预测异常 | 不同预测版本差异绝对值 `>=30%` |
| 周度销量波动异常 | 连续周实际销量偏离预估 `±20%` 以上，用于销售提示 |
| 冗余归因触发 | 仅 `medium/high/critical` 真实冗余进入冗余根因树；`low`/`预警监控` 只展示冗余依据，不计算冗余归因 |
| 物流延期 | 仅看未签收、预计签收日为今天或未来、计划签收日落在真实断货窗口内的批次；预计签收时间 - 计划签收时间 `>=7天` |
| 采购延期 | 仅看未签收、预计签收日为今天或未来、计划签收日落在真实断货窗口内的批次；物流商实际提货时间 - 计划发货时间 `>=4天` |

超卖/低卖差异率公式：

```text
差异率 = (实际销量或折算销量 - 2个月前预测值) / 2个月前预测值 * 100%
```

## 断货根因流程

```mermaid
flowchart TD
  A["发生断货"] --> B["取近3个月 / 近3周数据"]
  B --> C["用实际销量或折算销量对比2个月前预测值"]
  C --> D{"销售属性"}

  D -- "爆 / 旺" --> E{"差异率 > 20% ?"}
  D -- "平 / 滞" --> F{"差异率 > 10% ?"}
  D -- "其他 / 缺失" --> G["不判超卖"]

  E -- "是" --> H["输出：超卖导致断货"]
  E -- "否" --> I["未命中超卖"]
  F -- "是" --> H
  F -- "否" --> I
  G --> I

  I --> J{"供应链是否延期？"}
  H --> J
  J --> J1{"批次未签收，且预计签收在今天或未来？"}
  J1 -- "否" --> M["无供应延期根因"]
  J1 -- "是" --> J2{"计划签收日是否落在断货窗口？"}
  J2 -- "否" --> M
  J2 -- "是" --> J3{"延期是否达阈值？"}
  J3 -- "预计签收-计划签收 >=7天" --> K["输出：物流延期；带出延期备注"]
  J3 -- "物流商实际提货-计划发货 >=4天" --> L["输出：采购延期；带出延期备注"]
  J3 -- "未达阈值" --> M["无供应延期根因"]

  K --> N{"预测版本是否大幅变化？"}
  L --> N
  M --> N
  N -- "版本差异 >=30%" --> O["输出：销量预测异常"]
  N -- "未命中" --> P["无预测异常根因"]

  O --> Q{"是否已有任一根因？"}
  P --> Q
  Q -- "有" --> R["输出所有命中的根因"]
  Q -- "没有" --> S["输出：计划原因导致断货"]
```

断货会同时保留多个命中的根因，例如一个 SKU 可以同时输出：

```text
超卖导致断货
物流延期
采购延期
销量预测异常
```

只有当超卖、供应延期、销量预测异常都没有命中时，才输出：

```text
计划原因导致断货
```

## 冗余根因流程

```mermaid
flowchart TD
  A["命中冗余信号"] --> A1{"是否真实冗余？"}
  A1 -- "low / 预警监控" --> A2["只展示冗余依据，不计算冗余归因"]
  A1 -- "medium / high / critical" --> B["取近3个月 / 近3周数据"]
  B --> C["用实际销量或折算销量对比2个月前预测值"]
  C --> D{"销售属性"}

  D -- "爆 / 旺" --> E{"差异率 < -20% ?"}
  D -- "平 / 滞" --> F{"差异率 < -10% ?"}
  D -- "其他 / 缺失" --> G["不判低卖"]

  E -- "是" --> H["输出：低卖导致冗余"]
  E -- "否" --> I["未命中低卖"]
  F -- "是" --> H
  F -- "否" --> I
  G --> I

  I --> J{"实际销量是否不低于2个月前预测值？"}
  H --> K{"供应链是否延期？"}
  J -- "是" --> L["输出：供应异常导致冗余"]
  J -- "否" --> M["无供应异常根因"]

  L --> K
  M --> K
  K --> K1{"批次未签收，且预计签收在今天或未来？"}
  K1 -- "否" --> P["无供应延期根因"]
  K1 -- "是" --> K2{"计划签收日是否落在断货窗口？"}
  K2 -- "否" --> P
  K2 -- "是" --> K3{"延期是否达阈值？"}
  K3 -- "预计签收-计划签收 >=7天" --> N["输出：物流延期；带出延期备注"]
  K3 -- "物流商实际提货-计划发货 >=4天" --> O["输出：采购延期；带出延期备注"]
  K3 -- "未达阈值" --> P["无供应延期根因"]

  N --> Q{"预测版本是否大幅变化？"}
  O --> Q
  P --> Q
  Q -- "版本差异 >=30%" --> R["输出：销量预测异常"]
  Q -- "未命中" --> S["无预测异常根因"]

  R --> T{"是否已有任一根因？"}
  S --> T
  T -- "有" --> U["输出所有命中的根因"]
  T -- "没有" --> V["输出：计划原因导致冗余"]
```

只有当低卖、供应异常/延期、销量预测异常都没有命中时，才输出：

```text
计划原因导致冗余
```

## 数据来源关系

```mermaid
flowchart LR
  A["monthly_forecast_review.detail_monthly_totals"] --> B["月度超卖/低卖判断"]
  A --> C["2个月前预测值 forecast_version_totals[offset=2]"]
  D["monthly_forecast_review.forecast_versions[offset=2].weekly_estimates"] --> E["周度超卖/低卖判断"]
  F["weekly_estimates.actual_sales"] --> E
  G["first_leg_shipments / shipments / shipment_rows"] --> H["物流延期 / 采购延期判断"]
  I["forecast_anomalies / forecast_variance_checks"] --> J["销量预测异常判断"]
```
