## 1. 核心结论

这份材料讲的是跨境电商企业如何通过库存监控分析，打通销售计划、采购计划和发货计划，形成公司级的「一个计划」。

库存监控不是单纯看库存数量，而是围绕三个经营问题展开：

- 是否会断货。
- 是否有冗余库存。
- 什么时候应该补货。

对应的三个关键数据维度是：

- 库龄：反映存货过去的周转表现，主要用于识别冗余和仓储成本风险。
- 可售天数：反映现有库存面向未来销售的覆盖能力，主要用于判断冗余、断货和库存结构。
- 补货倒计时：反映现有库存距离必须补货的时间，主要用于判断断货风险和补货节奏。

## 3. 库存监控分析的作用

材料强调，一个错误的计划背后往往是库存监控分析缺失。这里的计划管理不只是采购计划和发货计划，还包括销售计划；三者需要通过流程拉通。

库存分析的管理价值包括：

- 通过海外库存可售天数分布识别异常库存，目标是让海外库存集中在约 60 天、全链路库存集中在约 120 天以内，并符合相对健康的分布。
- 对未来有断货风险的产品，及时通过加急交付、控销、调价、广告调整等方式减少断货损失。
- 精准识别哪些产品应该补货，减少补货不及时带来的断货损失，也减少不该补货带来的冗余损失。
- 通过库存销售货值、加权可售天数、预算目标、现金流之间的关系进行预警，避免「目标达成率低但备货 100%」或「低周转还继续加备货」这类风险。

## 4. 三个核心指标

### 4.1 库龄

库龄用于判断库存是否已经或即将产生额外仓储成本，尤其是亚马逊 FBA 的超期仓储费。

材料中的关键判断：

- 180 天以上开始关注超期仓储费。
- 270 天以上费用显著上升，是高风险库龄段。
- 365 天以上可能进入应弃置或批量清货的判断区间。

冗余判断不只看「卖得慢」，而是看继续正常销售、清货、批量清货、弃置之间，哪一种方式的利润或回款更优。

常见判断逻辑：

- 正常销售状态下的成本增量 >= 清货销售状态下的成本增量时，应考虑清货。
- 正常销售利润 <= 清货销售利润时，应考虑清货。
- 销售回款 <= 移除成本时，应考虑批量清货或弃置。
- 对薄利且 270 天以上的产品，要警惕沉没成本陷阱，核心问题是「还有没有回款」。

库龄数据可通过亚马逊《库存分类账》《库存分类账_详细视图》《库存分类账_一览视图》等数据生成库龄统计表和库龄 BI 看板。

### 4.2 可售天数

可售天数不是简单的「库存数量 ÷ 历史日均销量」。材料强调，计算可售天数应使用未来预估销量，并按库存分类、库存分级、共享库存关系进行计算。

可售天数的用途：

- 判断库存是否过多或不足。
- 判断不同仓段库存是否能接上。
- 支持补货倒计时计算。
- 支持营销侧和供应侧调整。

重点概念：

- 库存分类：FBA 库存、FBA 在途、海外仓库存、海外仓在途、本地仓库存、本地仓在途等。
- 库存分级：站点级、MSKU 级、SKU 级。
- 共享库存：同一 SKU 可能对应多个 MSKU 或多个站点，不能机械按单个 MSKU 看库存。

材料提醒：可售天数无法只用库存和历史销量线性推算，必须结合未来销售计划、生命周期、销售属性、产品属性和物流链路。

### 4.3 补货倒计时

补货倒计时用于判断「现在是否要补货」。

基础思路：

```text
补货倒计时 = 可售天数 - 安全库存天数 - 预估交付天数
```

其中：

- 安全库存天数是为了抵御销量、物流、生产不稳定保留的最低库存。
- 预估交付天数包括生产、出库、物流、入仓等链路时间。
- 如果补货倒计时 <= 0，通常意味着已经进入断货风险区，需要判断控销、加急、调拨、改物流方式或补货。

材料进一步区分不同链路下的补货倒计时，例如 FBA、海外仓、本地仓、直采直发、本地仓中转、海外仓中转、本地仓与海外仓双中转等模式。

## 5. 补货量与备货策略

材料反复强调：「何时补货」和「补多少货」必须分开。

常见补货量思路：

```text
备货量 = （安全天数 + 交付时长 + 补货频率） × 预估日销
```

但真实业务中需要继续修正：

- 补货倒计时是否为负。
- 本批次到货前是否会断货。
- 下一批次到货前是否能接上。
- 预估销量是否变化。
- 物流时效是否变化。
- 是否存在 MOQ、阶梯价、装柜率、供应商产能等约束。
- 是否会因为多采、早采导致库存冗余。

策略上：

- 新品销量不稳定，慢船会放大交付时长和备货量，通常更适合更快物流和多批次补货。
- 成熟期畅销品可以适当放大安全天数，使用更低成本物流。
- 衰退期或滞销品不能机械按历史销量补货，否则容易造成冗余。
- 供应商产能、旺季、高温限电、停工等也会影响是否早采、多采。

## 6. 营销侧与供应侧协同

材料把库存问题拆成营销侧和供应侧共同处理的问题。

营销侧可以做：

- 调整预估销量。
- 控销或限售。
- 调整价格、广告、活动。
- 对不同 MSKU、站点或渠道做上架数量调整。
- 对断货风险产品减少销量消耗。
- 对冗余产品加速清货。

供应侧可以做：

- 加急生产或加急物流。
- 调整采购计划。
- 调整发货计划。
- 调整物流渠道和发货模式。
- 调拨海外仓、本地仓、FBA 之间库存。
- 暂停采购、暂停发运或拦截在途计划。

材料强调：先有策略，后有计划。预估销量表只是营销体系输出之一，计划管理必须上通运营、下通供应。

## 7. 组织协同

库存监控分析不是某一个部门的事。

材料中涉及的典型分工：

- CEO：看问题、做跨部门决策、推动流程体系和组织能力优化。
- 运营：制定销售计划，输出预估销量，监控销量预实差，根据库存状态调整销售动作。
- 计划：监控预估销量和备货规则，统计断货与冗余损失，制定采购计划和发货计划。
- 采购：及时下单，缩短交期，提高准交率，反馈跟单结果。
- 物流：保证全链路库存准确性，及时出货，反馈物流跟单结果。

材料也提醒：流程体系不能代替组织建设。组织能力不足的问题，不能只靠流程表单解决。

## 8. 数字化与流程化

数字化的前提是业务标准化；业务标准化的前提是业务结构化、流程化；而流程化的前提是团队有专业知识储备。

材料中的数据和表单体系包括：

- 预估销量表。
- 库存分类与库存分级表。
- 库龄统计表。
- 库存分类账。
- 库存监控分析表。
- 利润测算表。
- 备货规则表。
- 发货计划表。
- 采购计划表。
- 物流跟单表。
- 营销调整单。
- 订单计划、待办事项、流程与数字化建设清单等。

核心是从业务流到单据流，再到数据流：

```text
销售计划 -> 预估销量 -> 库存监控分析 -> 营销/供应策略 -> 发货计划/采购计划 -> 跟单反馈 -> 再监控
```

## 9. 发货计划与采购计划

### 9.1 发货计划

发货计划本质是调拨计划，包括：

- 海外仓与海外仓之间调拨。
- 海外仓与 FBA 之间调拨。
- 本地仓与海外仓之间调拨。
- 本地仓与 FBA 之间调拨。

其目标是减少断货和冗余损失，缩短发货时间，降低物流成本。

发货计划需要判断：

- 是否需要发货。
- 发多少。
- 什么时候发。
- 用什么物流方式。
- 是否存在数量异常、时间异常、装柜异常、包装异常等。
- 谁审核，审核什么异常。

### 9.2 采购计划

采购计划的目标是减少断货和冗余损失、缩短采购下单时间、降低采购成本。

关键控制点包括：

- 是否选择加急生产或加急物流，应先在营销计划中讨论，因为会影响利润。
- 营销计划之后，计划部门需要判断断货风险和冗余风险。
- 数量异常包括 MOQ、阶梯价与冗余之间的冲突。
- 时间异常包括采购备货规则中的交付时间与营销计划时间之间的冲突。
- 审核本质上是审核异常，而不是重复走形式。

## 10. 平台费用规则的意义

材料通过亚马逊费用规则说明：平台收费规则背后体现平台战略和商家行为引导。

重点费用包括：

- FBA 配送费。
- 移除与弃置费。
- 仓储费。
- 仓储利用率附加费。
- 超期仓储费。
- 低库存水平费。
- 原包装折扣、SIPP 等。

对库存管理的启示：

- 平台费用变化会改变清货、移除、补货和仓储的经济账。
- 不理解费用规则，就无法判断库存冗余的真实成本。
- 库龄、可售天数和平台费用必须放在一起算利润和现金流。

## 11. 实操落地建议

1. 先建立统一的库存口径：仓段分类、SKU/MSKU/店铺分级、共享库存关系必须清楚。
2. 用未来预估销量计算可售天数，不要只用历史平均销量。
3. 每个产品都要有安全库存天数、交付天数、补货频率等备货规则。
4. 补货倒计时 <= 0 的产品进入断货风险池，区分轻微、中等、严重风险。
5. 库龄 180+、270+、365+ 的库存进入冗余风险池，结合费用和利润测算处理动作。
6. 对断货风险先判断营销侧能否控销，再判断供应侧能否加急或调拨。
7. 对冗余风险先判断是否清货、批量清货、弃置或暂停采购/发运。
8. 把营销计划、发货计划、采购计划放在同一套流程里，形成闭环。
9. 用数据表和流程表支持判断，但不要让表单替代组织决策。

## 12. 一句话复盘

库存监控分析的本质，是用库龄、可售天数和补货倒计时，把「卖什么、卖多少、什么时候补、补多少、用什么方式补」变成跨部门共同维护的一套经营计划。
