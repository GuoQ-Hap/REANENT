数据来源

本地仓 / 海外仓库存
dwd_lingxing_inventory_details，取表内最新 date，只取 product_total > 0 的行。
FBA库存
dwd_lingxing_fba_warehouse_detail，取表内最新 date，只取 total > 0 的行。
FBA台账损耗
dwd_lingxing_inventory_ledger_summary_incr，取表内最新 date，只取 lost / damaged / disposed 非 0 的行。
仓库维表
dwd_lingxing_sc_warehouse，取最新日期后按 wid 关联仓库名称、仓库类型、国家。

单位成本

本地仓 / 海外仓
单位成本优先级：stock_price → stock_cost → purchase_price → stock_cost_total / product_total。
FBA
FBA按每个状态自己的金额字段反推：状态金额 / 状态数量。例如 afn_fulfillable_quantity_price / afn_fulfillable_quantity。
成本缺失
数量大于 0 且 unitCost = 0 且 amount = 0 时标记为“成本缺失”，不静默当作真实 0 成本。

占用资金

本地仓 / 海外仓金额
状态数量 × 单位成本。状态包括：可用 product_valid_num、锁定 product_lock_num、次品 product_bad_num、待检 product_qc_num、调拨在途 product_onway。
FBA金额
直接取每个状态金额字段。状态包括：FBA可售、待调仓、调仓中、待发货、FBM可售、FBA不可售、计划入库、在途、入库中、调查中。
总资金占用
当前筛选范围内所有占用明细的 amount 求和。
预计仓储费
FBA按金额占比分摊：estimated_storage_cost_next_month × 状态金额 / total_price；本地仓/海外仓当前为 0。

资金结构

正常周转资金
状态包含“可售”或“可用”，且不命中风险占用、在途资金的明细金额。
风险占用资金
命中任一条件：成本缺失、库龄 >= 181 天、状态包含“不可售/次品/待检/锁定/预留/调查/待发货”。
在途资金
状态包含“在途/入库/计划/待调仓/调仓”的明细金额。
风险标签
优先级：成本缺失 → 365天+ → 181天+ → 不可售风险 → 冻结占用 → 在途资金 → 正常周转。

损耗现金

本地仓 / 海外仓静态损耗
次品损耗：product_bad_num × 单位成本，动作标记为“降本复核”。
FBA不可售损耗
取 afn_unsellable_quantity_price 作为库存成本；单位成本为 afn_unsellable_quantity_price / afn_unsellable_quantity，动作标记为“移除测算”。
台账流水损耗
丢失、损毁、弃置数量来自 dwd_lingxing_inventory_ledger_summary_incr；单位成本优先用当前占用明细中相同 SKU/FNSKU 的成本。
净损失
成本口径：库存成本 + 处理费用；回款口径：库存成本 + 处理费用 - 预计回款。当前处理费用和预计回款字段暂为 0。

页面规则

筛选范围
日期、仓库类型、国家、SKU/FNSKU/仓库搜索都会同时作用于占用明细和损耗明细。
资金分布
按仓库类型 type 汇总 amount，展示各类型资金占比。
图表钻取
资金路线节点、状态拆分、来源拆分、关注拆分都可点击筛选；再次点击同一条件或点击“清除”恢复全部。
处理队列
按净损失从高到低排序，展示前 8 条损耗项；净损失 >= 450 为高，>= 180 为中，其余为低。
导出
导出当前筛选后的占用明细：日期、仓库、类型、国家、SKU、FNSKU、状态、数量、单位成本、占用金额。
